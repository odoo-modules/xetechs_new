Start Stop Pause functions for Project Task

Add Start Stop and Pause buttons in Kanban and Form view with the preservation of working time in the timesheets, and use the Fast Timesheet

DEMO: http://demo.appstg.com, LOGIN demo PASSWORD demo

To properly save timesheets, the user needs to be an Employee. You can not run the task if the user is not an employee. In the module there is an auto refresh mechanism Kanban and Form view. If one of the users in your browser launches or stops the task - in a few seconds it will be seen by other users if they open this view.



One user can not start more than one task at a time. If the user attempts to start another task, the system prompts you to pause the running task. The Project Manager can run tasks assigned to himself and other users. The Project User can only run his own tasks. If an unassigned task is started, it is automatically assigned to the runner.


When you click on the Stop button, a window appears where you need to briefly describe the work done.


After press Submit button the task creates timesheet, how long the given employee worked.


Project User can close only its task, Project Manager can close any tasks.

For use Fast Timesheet setup default Project for registration timesheet


If you start Fast Timesheet, another running task will automatically pause and be remembered. Fast timesheet don't paused


After press Stop button&nbsp;You will be asked to write a job description and select the project to which to attach the timesheet. If the Project field is left blank, the timesheet will be attached to the Default Project. If you leave a tick in the field "Do you want to start the paused task?", Then the paused task will automatically start.
